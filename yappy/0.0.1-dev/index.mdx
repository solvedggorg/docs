**yappy** is a Linux-only native desktop shell written in Zig. It targets Electron / Tauri / Deno Desktop class apps **without** an embedded browser: UI is **Qt and GTK**, and app logic runs on **[scripty](/scripty)** (JS runtime + project manager).

<Warning>
  **Permanently Linux-only** and **always native UI** — no webview, no Chromium, no HTML-as-shell.
</Warning>

## What yappy is for

- **Native Linux UI** — first-class **Qt** and **GTK** backends (v0.1.0 bar: both)
- **scripty for logic** — one JS/TS runtime and PM embedded in the host; no Node main process
- **Capability-scoped host** — FS and related APIs default deny; grants are explicit
- **Backend-agnostic app API** — apps target a unified widget surface where possible

## Product split

| Layer | Owner | Responsibility |
| --- | --- | --- |
| **scripty** | [scripty](/scripty) | JS VM, host ABI, package manager, embed API |
| **yappy** | this product | Process main, event loop, **native UI**, OS integration, capabilities, packaging |
| **App code** | app repo | JS/TS business logic via scripty; UI via yappy native APIs |

Rules:

1. No browser stack in yappy.
2. No second JS engine — only scripty.
3. Package management is scripty’s; yappy may invoke it but does not reimplement npm.
4. Native APIs are yappy’s product surface.

## Architecture (target)

```text
yappy process (Linux)
  main / CLI · capability host · backend selector (qt | gtk)
           │
           ▼
  unified widget + window + event API
           │
           ▼
  scripty embed  (isolates · modules · PM graph)
    app JS/TS  ←→  yappy_* host functions
```

## Status

yappy is in **early access / scaffold**. Install paths, CLI reference, and the native API matrix will land as the binary stabilizes. Backend and API matrices currently mark required rows as **unsupported** until implementation lands — that honesty is intentional.

| Decision | Status |
| --- | --- |
| Linux-only forever | Accepted |
| Native UI only (no webview) | Accepted |
| scripty as the only JS runtime | Accepted |
| v0.1.0 = both Qt and GTK first-class | Accepted |
| Default backend when both installed | Pending |
| Qt binding approach / GTK4 pin | Spike / proposed |
