**scripty** es un entorno de ejecución de JavaScript y un gestor de proyectos escrito en Zig. Esta herramienta pretende cubrir las API y los flujos de trabajo que ya usas con Node.js, Deno y Bun —con formateo y linting integrados— sin tener que apilar tres entornos de ejecución distintos.

## Para qué sirve scripty

- **Un único entorno de ejecución** para las API de Node, Deno y Bun (el objetivo del producto es la compatibilidad total; la implementación aún está en una fase temprana)
- **Gestión de proyectos** para árboles JS/TS: flujos de trabajo de instalación, ejecución y espacio de trabajo, todo en un solo CLI
- **Formateo y linting integrados** con reglas y compatibilidad con el ecosistema, para que no tengas que montar una cadena de herramientas aparte para cada repositorio
- **La velocidad como objetivo prioritario**: el objetivo de marketing es del orden de **100× Bun** para las rutas en torno a las que se ha diseñado scripty

## Cómo encaja en la suite

scripty se sitúa junto a [rusty](/es/rusty) (Rust), [worgo](/es/worgo) (Go), [pyppi](/es/pyppi) (Python) y [yappy](/es/yappy) (host de escritorio nativo): herramientas de [solved.gg](https://solved.gg).

| Hermano | Ecosistema |
| --- | --- |
| [rusty](/es/rusty) | Gestor de paquetes y cadena de herramientas de Rust (Linux) |
| **scripty** | Entorno de ejecución de JavaScript / TypeScript + PM |
| [worgo](/es/worgo) | Gestor de proyectos del espacio de trabajo Go |
| [pyppi](/es/pyppi) | Python gestor de paquetes + herramienta de proyectos |

| [git](/es/git) | solved.gg servidor de git UI |
| [depotsdk-go](/es/depotsdk-go) | Go SDK para el Depot API |
