**scripty** is a JavaScript runtime and project manager written in Zig. One tool aims to cover the APIs and workflows you already use with Node.js, Deno, and Bun — with built-in formatting and linting — without stacking three separate runtimes. The engine is an **own VM** (not V8, JSC, or SpiderMonkey).

<Warning>
  **Linux only, permanently.** First-party builds, CI, and dogfood are Linux-only. Desktop UI never lives in scripty (no webview / Chromium) — hosts such as [yappy](/yappy) embed scripty from separate repositories.
</Warning>

## What scripty is for

- **One runtime surface** for Node, Deno, and Bun APIs (full compatibility is the product goal; the implementation is still early)
- **Own Zig VM** — parse → bytecode → interpret; TS type strip + JSX transform by extension
- **Project management** for JS/TS trees: install, run, and workspace workflows in one CLI (maturing with the engine)
- **Speed as a first-class goal** — marketing target is on the order of **100×** the fastest comparable baseline *on paths scripty designs for* (define benches; not universal web parity)

## CLI surface (today)

```bash
scripty run <file.js|file.ts|file.tsx|file.jsx>
scripty <file.js|file.ts|file.tsx|file.jsx>   # same as run
scripty test262 [--path …] [--feature …] [--report …] [--limit …]
scripty --help
```

Version line: `scripty v0.0.0` during pre-v0.0.1. No Node/Deno/Bun fallback.
