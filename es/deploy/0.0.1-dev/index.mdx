---

**deploy** es un Zig CLI que conecta **Next.js → Cloudflare Workers** cuando **creas o adoptas** una app. Reescribe las configuraciones, aplica parches a las dependencias, genera wrangler y lo envía a través de un entorno de ejecución de Workers **propio** bajo `@solvedgg/deploy-*`, no mediante un adaptador OpenNext posterior a la compilación.

## Para qué sirve deploy

- **Conexión de Cloudflare en el momento de la inicialización** — `deploy create` / `deploy init` en lugar de añadir un adaptador a posteriori
- **Verbos al estilo de Vercel** — `deploy`, `env`, `link`, `whoami`, `logs`, `dev`, `check`
- **Tiempo de ejecución y compilación propios** — `@solvedgg/deploy-runtime` y `@solvedgg/deploy-build` (código original, no un cambio de marca)
- **Compatibilidad real** — compatible / parcial / no compatible, todo ello registrado en una matriz actualizada; las funciones no compatibles fallan de forma cerrada a menos que las fuerces

## Cómo se corresponde

| Necesidad | Cloudflare |
| --- | --- |
| Servidor de aplicaciones | Trabajadores |
| Recursos estáticos | Recursos de los trabajadores |
| KV / caché perimetral | KV + caché API |
| Blobs | R2 |
| SQL | D1 / Hyperdrive |
| Imágenes | Cloudflare Imágenes / transformación |
| Cron | `triggers.crons` |
| IA | Workers AI |

## Instalación (desde el código fuente)

Requiere [Zig](https://ziglang.org/) `0.16+`.

```bash
git clone https://github.com/solvedggorg/deploy.git
cd deploy
zig build
./zig-out/bin/deploy help
```

## Inicio rápido

```bash
# greenfield
deploy create my-app --yes --name my-app
cd my-app
bun install   # link workspace packages or published @solvedgg/deploy-*

# existing Next app
deploy init . --name my-app
deploy check
deploy link --name my-app
deploy deploy
```

## Superficie CLI

| Comando | Propósito |
| --- | --- |
| `deploy create [dir]` | Estructura básica del enrutador de aplicaciones Next + configuración de CF |
| `deploy init [dir]` | Adoptar una app Next ya existente |
| `deploy check` | Análisis de compatibilidad |
| `deploy link` | Escribir `deploy.json` (+ wrangler si falta) |
| `deploy deploy` | Compilar + `wrangler deploy` |
| `deploy dev` | Ruta local de `wrangler dev` |
| `deploy env …` | Puente de secretos/variables |
| `deploy whoami` | Identidad de Cloudflare |
| `deploy logs` | `wrangler tail` |
| `deploy matrix` | Mostrar estimación de cobertura |

Indicadores comunes: `--force`, `--yes`, `--name`, `--account-id`, `--rewrite-next-config`.
