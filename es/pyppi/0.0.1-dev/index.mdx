**pyppi** es un gestor de paquetes y proyectos Python escrito en Zig. Se suma a [rusty](/es/rusty), [scripty](/es/scripty) y [worgo](/es/worgo) dentro del conjunto de herramientas de lenguaje solved.gg: una única estructura CLI para los flujos de trabajo de instalación, resolución y gestión de proyectos, en lugar de tener que apilar comandos ad hoc como pip, poetry, uv o virtualenv en cada repositorio.

## Para qué sirve pyppi

- **Python gestión de proyectos**: inicializar, instalar, bloquear y ejecutar con una única CLI instrucción
- **Implementación de Zig**: el mismo enfoque sistémico que las herramientas de otros lenguajes similares: comportamiento explícito, rutas rápidas y trámites mínimos
- **Acceso anticipado honesto**: la interfaz pública de CLI aún se está desarrollando; no des por hecho que ya haya plena compatibilidad con pip o uv

## Cómo encaja en el conjunto

| Hermano | Ecosistema |
| --- | --- |
| [rusty](/es/rusty) | Gestor de paquetes y cadena de herramientas de Rust (Linux) |
| [scripty](/es/scripty) | Entorno de ejecución de JavaScript / TypeScript + PM |
| [worgo](/es/worgo) | Gestor de proyectos del espacio de trabajo Go |
| **pyppi** | Gestor de paquetes + herramienta de proyectos Python |

| [deploy](/es/deploy) | Next.js → Cloudflare Workers CLI |
| [git](/es/git) | solved.gg servidor de git UI |
| [depotsdk-go](/es/depotsdk-go) | Go SDK para el Depot API |
