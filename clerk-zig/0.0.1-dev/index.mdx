**clerk-zig** is a Zig package that centralizes **solved.gg / iResolved product identity** for the Project Managers suite (rusty, scripty, hasky, deploy, …). It is **not** a full Clerk SDK: no Backend API, no JWKS middleware, no webhooks.

## What clerk-zig is for

- **Clerk OAuth** for product CLIs: authorize, token exchange, refresh, revoke, userinfo
- **PKCE + loopback callback + browser open** for interactive `auth login`
- **Shared session store** at **`$PMS_HOME/auth/session.db`** (via [zig-libsql](/zig-libsql))
- **Shared suite root** **`$PMS_HOME`** (default `~/.pms`) so one login serves every PM

<Warning>
  **Linux only.** First-party builds and session semantics target Linux. Do not expect non-Linux support.
</Warning>

## Architecture

```text
rusty / scripty / hasky CLI  (auth login|…)
              │  @import("clerk_zig")
              ▼
         clerk-zig
    config → pkce → oauth ⇄ Clerk (https issuer)
             callback + browser
             login orchestration
             store ──► zig-libsql
             paths ($PMS_HOME)
              │
              ▼
    $PMS_HOME/auth/session.db
```

## Environment

| Variable | Role |
| --- | --- |
| `PMS_HOME` | Suite data root (default `~/.pms`) |
| `PMS_AUTH_CLIENT_ID` | Clerk OAuth **public** client id (required for login) |
| `PMS_AUTH_ISSUER` | Clerk Frontend API / issuer URL (`https://` required) |
| `PMS_AUTH_SCOPES` | Optional scopes (default: `profile email offline_access`) |
| `RUSTY_AUTH_*` | **Legacy aliases** used when the matching `PMS_*` value is unset |

## Product CLI surface (consumers)

Keep brand-facing commands in each PM:

```bash
rusty auth login     # browser PKCE → $PMS_HOME/auth/session.db
rusty auth logout
rusty auth whoami
rusty auth status    # path + session metadata, no secrets
```

Never print access or refresh tokens.

## Build & consume

Requires Zig **0.16.x**, Linux, and **zig-libsql**.

```bash
# library checkout
zig build
zig build test
```

See [Consuming](/clerk-zig/0.0.1-dev/consuming) for `build.zig.zon` wiring and [\$PMS_HOME layout](/clerk-zig/0.0.1-dev/pms-home) for the suite directory contract.
