title: "$PMS_HOME layout"
description: "Unified suite data root for PMS toolchains — auth, optional registry, per-PM trees."
---

Unified data root for PMS toolchains. Default: **`~/.pms`**. Override with **`PMS_HOME`**.

```text
$PMS_HOME/
  auth/
    session.db              # clerk-zig session (0600, DELETE journal)
  toolchains/
    manifest.toml           # optional: registered PM installs (later)
  bin/                      # optional: unified PATH shims (later)
  rusty/                    # migrates from ~/.rusty (per-PM data)
    toolchains/
    store/
    …
  hasky/                    # migrates from ~/.hasky
  scripty/
  deploy/
  …
```

## Ownership

| Path | Owner package |
| --- | --- |
| `$PMS_HOME` resolution | **clerk-zig** `paths` (shared) |
| `$PMS_HOME/auth/**` | **clerk-zig** only |
| `$PMS_HOME/toolchains/manifest.toml` | **clerk-zig** registry API (optional later) |
| `$PMS_HOME/<pm>/**` | that PM’s own paths module |

## Migration notes

- rusty historically used `$RUSTY_HOME` / `~/.rusty` and `auth.db` at its root.
- After cutover: the product session lives only at **`$PMS_HOME/auth/session.db`**.
- One-time migrate: if old `$RUSTY_HOME/auth.db` exists and the new session is empty, copy or import.
- Per-PM toolchains and stores move under `$PMS_HOME/<pm>/` in follow-up work; auth does not block that migration.

## Security

- `auth/session.db` mode **0600**
- No world-readable tokens
- Prefer `PRAGMA journal_mode=DELETE` (no WAL sidecars that need separate permissioning)
