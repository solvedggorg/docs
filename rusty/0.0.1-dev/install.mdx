rusty ships as a standalone CLI. This page covers Linux nightly install and layout after you have a binary.

<Warning>
  rusty runs on **Linux only** (x86_64 and aarch64). Do not install or expect support on macOS or Windows.
</Warning>

## Install (Linux nightly)

Installs the newest published `nightly-<sha>` release binary to `$RUSTY_HOME/bin/rusty` (default `~/.rusty/bin/rusty`) and symlinks `~/.local/bin/rusty` → that binary. Verifies `SHA256SUMS`.

```bash
# from a checkout (recommended while the repo is private)
./install.sh

# or fetch the script via gh, then run (needs `gh auth login`)
gh api repos/solvedggorg/rusty/contents/install.sh?ref=master \
  --jq .content | base64 -d | bash
```

Auth for private release assets: **`gh` logged in** (preferred) or **`GITHUB_TOKEN`**. Public anonymous download works only if the repo/releases become public.

Override paths or pin a tag:

```bash
RUSTY_HOME=~/.rusty RUSTY_LINK_DIR=~/.local/bin ./install.sh
RUSTY_NIGHTLY_TAG=nightly-<sha> ./install.sh
```

Ensure `~/.local/bin` is on your `PATH`.

## Build from source

Requires Zig **0.16.x** and Linux:

```bash
zig build
./zig-out/bin/rusty --version
```

## Data directory: `RUSTY_HOME`

| Path | Role |
| --- | --- |
| `$RUSTY_HOME` (default `~/.rusty`) | Toolchains, CAS store, shims |
| `$RUSTY_HOME/bin` | Active toolchain proxies (`rustc`, …) and `rusty-wrapper` |
| `$RUSTY_HOME/cargo-home` | Managed layout when cargo-compatible env is injected |

```bash
export RUSTY_HOME="$HOME/.rusty"   # optional; this is already the default
```

Suite-wide product auth does **not** live under `$RUSTY_HOME` anymore — see below.

## First toolchain

```bash
rusty env install stable
rusty env use stable
rusty env list
rusty env show
rusty env which rustc
```

Install extra targets or components as needed:

```bash
rusty env install stable x86_64-unknown-linux-musl
rusty env install stable clippy
rusty env install stable rustfmt
# cargo is opt-in if you need interop verbs (doc/bench/tree):
rusty env install stable cargo
```

Without cargo on PATH, interop commands (`doc`, `bench`, `tree`) fail closed (exit **127**). **`clippy` and `fmt` are native** and do not require cargo.

## PATH for shims

After `rusty env use <id>`, put shims first:

```bash
export PATH="${RUSTY_HOME:-$HOME/.rusty}/bin:$PATH"
```

Override the active toolchain for a single command with `+toolchain` or `RUSTY_TOOLCHAIN`.

## Product authentication (optional)

Product identity is **not** crates.io login. Clerk-based sign-in is implemented via **[clerk-zig](/clerk-zig)** and stores a shared suite session under **`$PMS_HOME`** (default `~/.pms`):

```bash
# Requires PMS_AUTH_CLIENT_ID and PMS_AUTH_ISSUER
# (RUSTY_AUTH_* legacy aliases still accepted)
rusty auth login
rusty auth whoami
rusty auth status
rusty auth logout
```

Session path: `$PMS_HOME/auth/session.db`. See [\$PMS_HOME layout](/clerk-zig/0.0.1-dev/pms-home).

Authenticated [solved.gg](https://solved.gg) / iResolved accounts may use rusty under the product EULA with payment portions of the source-available license bypassed — see the license shipped with your build.

## Project-local layout

Per repository, rusty uses:

```text
.rusty/
  format          # "rusty" or "cargo"
  target/         # build outputs (also CARGO_TARGET_DIR during builds)
  …               # lock digests, layer index, graph cache, …
```

You normally do not create this by hand; `rusty init` and other commands manage it.
